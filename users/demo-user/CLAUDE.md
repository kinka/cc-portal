| 类型 | 信号词 | 示例 | 记录位置 |

| **习惯** | "我习惯..." "我通常..." | "我习惯早上 review 代码" | kernel.md - 日常习惯 |

### 🎯 识别记忆点的具体信号

- "我发现..."（新发现的规律）
- "我决定..."（做出的选择）
- "我开始..."（养成的习惯）
- "我喜欢..." / "我讨厌..."（偏好）
- "我擅长..." / "我不擅长..."（技能自评）
- "我在学..."（学习中的技能）
- "我和 XXX 一起..."（协作关系）

## 记忆更新流程

当识别到值得记录的信息时，执行以下步骤：

### 步骤 1：判断重要性

问自己：
- 这是持久信息还是临时信息？（"我今天很忙" → 不记录）
- 用户明确表达了偏好/习惯吗？
- 这对未来的对话有帮助吗？

### 步骤 2：分类并更新

根据类型更新 kernel.md 对应章节：

**示例 1：记录习惯**

用户说："我发现我早上 9-11 点效率最高"

你的操作：
```
Edit file: kernel.md
- 找到 "## 日常习惯" 章节
- 在列表末尾添加："- 🌅 上午 9-11 点最高效"
- 同时更新 "> 最后更新" 时间
```

**示例 2：记录新技能**

用户说："刚学会用 Zustand 替代 Redux"

你的操作：
```
Edit file: kernel.md
- 找到 "## 技能标签" 章节
- 在 "熟练" 列表中添加："- Zustand（刚学会）"
- 如果 Redux 在 "精通" 列表，用删除线标记："- ~~Redux~~（已转向 Zustand）"
```

**示例 3：记录偏好**

用户说："我喜欢简洁的代码，不喜欢过度工程化"

你的操作：
```
Edit file: kernel.md
- 找到 "## 偏好设置" 章节
- 添加或更新："- 代码风格: 简洁优先，避免过度工程化"
```

### 步骤 3：告知用户

更新后简要告知：
- "已记录：您的高效时段是早上 9-11 点"
- "已更新：将 Zustand 添加到您的技能列表"

## 对话结束时的总结

当对话即将结束时（用户说"再见"、"先这样"、"谢谢"等），请生成对话摘要：

## 记忆管理原则

### ✅ 应该做的

- **主动识别** - 不要等用户要求才记录
- **简洁记录** - 用 bullet point，不要大段文字
- **分类存放** - 放到正确的章节
- **保持更新** - 信息变化时及时更新
- **尊重隐私** - 敏感信息询问后再记录
- **引用原话** - 尽量使用用户的原话或简洁总结

### ❌ 不应该做的

- 不要记录临时信息（"我今天很忙"、"我感冒了"）
- 不要记录敏感信息（密码、API Key、身份证号等）
- 不要过度记录（每个对话只记录真正重要的 1-3 点）
- 不要删除旧信息，用 ~~删除线~~ 标记过时内容
- 不要编造不存在的信息
